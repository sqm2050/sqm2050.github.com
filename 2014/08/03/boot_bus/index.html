概览

boot bus 用途：The boot bus hardware provides a general-purpose bus that typically attaches to novolatile devices, like flash memories(both NOR and NAND), compact flash memeories,and ROMs,and can service the initial boot address.

boot bus提供一个连接非易失设备如flash设备(NOR和NAND)，CF卡和ROMs的通用总线，而且支持初始化启动地址。

大端boot bus包含：

  • 八个片选信号

  • 32位地址/数据线

  • 可编程8/16位字长

    • 在8位操作下，BOOT_AD<31:24>是数据总线

    • 在32位操作下，BOOT_AD<31:24>是数据总线高位字节，BOOT_AD<23:16>是数据总线地位字节

boot bus硬件还包含两个本地的128字节的内存区域。

地址

boot bus占用物理地址4GB的空间，范围是：0x1 0000 0000 0000到0x1 0000 FFFF FFFF。

内核会转换物理地址范围是0x0 0000 1000 0000 - 0x0 0000 1fff ffff到boot bus的物理地址范围是0x1 0000 1000 0000 - 0x1 0000 1fff ffff。

如果一个系统内没有任何设备链接到boot bus，则boot bus内的两个本地的内存区域一定要提供表中(即0x1 0000 1FC0 0000和0x1 0000 1FC0 0480)异常的处理指令。

每个本地内存区域可以保存32个指令，如果需要更多的指令去处理异常，异常捕获必须在32个指令内掉转到L2/DRAM中。

地址匹配和区域

boot bus硬件仅提供对匹配的物理的地址的访问。首先会匹配两个本地内存区域，未名中，则再拿物理地址匹配总线区域。

本地内存区域

MIO_BOOT_LOC_CFGn[EN]如果被设置了，则代表了对应本地内存区域被使能。如果PA<31:7>等于MIO_BOOT_LOC_CFGn[BASE]，就是匹配了本地内存区域。

本地内存在使用之前必须被软件填充。填充本地内存区域0方法：

  • 设置MIO_BOOT_LOC_ADR为0x0(本地内存区域1为0x80)

  • 写MIO_BOOT_LOC_DAT为16次，每次8个字节

总线区域

8个总线区域通过MIO_BOOT_REG_CFGn[EN]激活，每个总线区域的大小是(MIO_BOOT_REG_CFGn[SIZE] + 1) * 2^16。

匹配条件：

PA<31:16> >= MIO_BOOT_REG_CFGn[BASE] AND
PA<31:16> <= MIO_BOOT_REG_CFGn[BASE] + MIO_BOOT_REG_CFGn[SIZE]

重置配置和启动

复位之后，总线区域0设置如下：

  • 是唯一有效区域

  • 是最大值

  • 映射物理地址0x1 0000 1FC0 0000到连接片选为0的设备的0x0地址

  • 使用8位数据宽度

  • 操作是独立的

这表示当设备自启动时，连接到片选为0的设备的0地址必须包含设备的启动代码。当设备启动时，0核立即执行这里的代码，紧随复位之后。
